# SDD 质量检查清单

## Proposal Gate
- [ ] 当前痛点有数据或用户反馈支撑
- [ ] 预期收益可衡量
- [ ] 影响范围完整（所有涉及模块/服务）
- [ ] 约束条件已列出
- [ ] Non-Goals 明确
- [ ] 团队确认方向 OK

## Specs Gate
- [ ] 每个 REQ 编号唯一（REQ-001, REQ-002...）
- [ ] 优先级标注（P0/P1/P2）
- [ ] 每个需求有 GWT 场景
- [ ] 边界条件已覆盖：空输入、非法输入、超时、并发
- [ ] 非功能需求已列出（性能/安全/兼容性）
- [ ] MUST/SHALL 关键字使用正确

## Design Gate
- [ ] 架构决策有选型理由 + 否决方案理由
- [ ] 系统架构图完整
- [ ] 数据流含正常路径 + 异常路径
- [ ] 新增/修改/排除文件清单完整
- [ ] 数据模型变更含 UP + DOWN
- [ ] 风险评估完整（概率 x 影响 x 缓解）
- [ ] 回滚方案可执行
- [ ] 监控指标明确

## Tasks Gate
- [ ] 每个任务 INVEST（独立/可协商/有价值/可估/小/可测）
- [ ] 依赖图正确
- [ ] 每个任务有验证方式
- [ ] 总估时合理（单个 Phase < 2周）
- [ ] 完成标准明确

## 常见反模式
- ❌ proposal 只写"要做X"，没写"为什么做"
- ❌ specs 缺少 GWT 场景，只有功能列表
- ❌ design 只有方案没有备选
- ❌ tasks 没有验证方式
- ❌ 缺少 Non-Goals
- ❌ 数据库变更没有 DOWN 脚本
